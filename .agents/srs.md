# Software Requirements Specification

## Purpose

Defines what the system must do.

Never describe implementation.

---

# Functional Requirements

Authentication

Users shall login.

---

Inventory

Users shall manage products.

---

Sales

Users shall create sales.

---

Customers

Users shall manage customers.

---

Dashboard

Users shall view analytics.

---

Offline

System shall continue working without internet.

---

Synchronization

System shall synchronize automatically.

---

Barcode

System shall scan products.

---

Reports

System shall generate reports.

---

Security

Users shall only access company data.

---

Non Functional Requirements

Availability

Offline capable.

Performance

UI updates below 100ms.

Scalability

Support multiple companies.

Maintainability

Feature based modules.

Security

Role based authorization.

---

Constraints

Firestore

React

Capacitor

---

Acceptance Criteria

Each feature shall have

- tests

- validation

- error handling
